    Configuration

    git config --global -l

    git config --global user.name "John Doe"

    git config --global user.email "johndoe@example.com"

    git config --global push.default current

    git config --global merge.defaultToUpstream true

    SSH Key Configuration

    bash -c "$(curl -fsSL http://bit.ly/pcs-git)"

    This copies something for you (as if you had typed ⌘C) that will allow you to magically access GitHub repositories without passwords.

    We'll paste that into the GitHub SSH keys with ⌘V.

    A Solo Workflow

    1. create a new Github repo
    2. clone the repo from github to our machine
    3. do some work
    4. add files to track
    5. commit changes and write a commit message
    6. Push our changes up to Github
    7. Repeat steps 3-6

    A Solo Workflow (including commands)

    1. create a new Github repo
    2. clone the repo from github to our machine git clone git@github.com:user/repository.git
    3. do some work
    4. add files to track: git add index.html
    5. commit changes and write a commit message: git commit -m "make a fancy webpage"
    6. Push our changes up to Github git push
    7. Repeat steps 3-6

    Challenge

    Following the steps from the previous slide:

    • Create an index.html file and put some code in it
    • Get the new file up on github
    • When you are done and have achieved victory, say "woot!"

    A Collaborative Workflow: Part 1

    1. create a new Github repo
    2. add some collaborators (admin only)
    3. clone the repo from github to our machine git clone git@github.com:user/repository.git
    4. Make a working/feature branch: git checkout -b branchname
    5. do some work

    A Collaborative Workflow: Part 2

    1. add files to track: git add fileName.html
    2. commit changes and write a commit message: git commit -m "make a fancy webpage"
    3. Push our changes up to Github git push
    4. do a pull request
    5. merge the pull request (admin only)

    Challenge

    Following the steps from the livious slides:

    1. Get into groups of 3
    2. Elect an admin
    3. The admin will make a repo on github and add collaborators
    4. each person in the group will work in the own branch named with their own name
    5. each person will make an html file named with their own name
    6. work in your own branch and then get everything up on github
    7. everyone does a pull request, then ONLY the admin merges
    8. lastly, do a git pull to update your local master branch
    9. When you are done and have achieved victory, say "woot!"
