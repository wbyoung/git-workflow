    A Solo Workflow

    1. create a new Github repo
    2. clone the repo from github to our machine
    3. do some work
    4. add files to track
    5. commit changes and write a commit message
    6. Push our changes up to Github
    7. Repeat steps 3-6

    A Solo Workflow (including commands)

    1. create a new Github repo
    2. clone the repo from github to our machine 
      git clone git@github.com:user/repository.git
    3. do some work
    4. add files to track: 
      git add index.html
    5. commit changes and write a commit message: 
      git commit -m "make a fancy webpage"
    6. Push our changes up to Github 
      git push
    7. Repeat steps 3-6

    Challenge

    A Collaborative Workflow: Part 1

    1. create a new Github repo
    2. add some collaborators (admin only)
    3. clone the repo from github to our machine 
      git clone git@github.com:user/repository.git
    4. Make a working branch (feature branch) 
      git checkout -b branchname
    5. do some work

    A Collaborative Workflow: Part 2

    1. add files to track 
      git add fileName.html
    2. commit changes and write a commit message: 
      git commit -m "make a fancy webpage"
    3. Push our changes up to Github 
      git push
    4. do a pull request
    5. merge the pull request (admin only)

    Challenge

    Following the steps from the previous slides:

    • Get into groups of 3
    • Elect a supreme leader / admin
    • The leader will make a repo on github and add collaborators
    • each person in the group will work in the own branch named with er... their name
    • each person will make an html file named with their name
    • work in your own branch and then get everything up on github
    • everyone does a pull request, then admin merges
    • lastly, do a git pull to update your local master branch
    • When you are done and have achieved victory, say "woot!"
